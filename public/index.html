O que é o Laracol?

Laracol é uma adaptação do Laravel para COBOL baseada em três princípios: Simplicidade, Produtividade e Modernidade. Padrões de frameworks web modernos implementados em GnuCOBOL com suporte a MVC, Roteamento REST, Middleware e Models.

📂
Estrutura MVC
Separação clara de responsabilidades com Controllers, Models e Views organizados.
⇄
Roteamento Flexível
Defina endpoints GET, POST, PUT e DELETE com mapeamento direto para Controllers.
🛡
Middleware
Autenticação, validação e CORS. Processe requisições antes dos Controllers.
🗄
Models & Data
Padrão Active Record para CRUD. Suporte a banco de dados, migrations e seeders.
💡

Arquitetura Baseada em CGI: Laracol implementa o padrão MVC sobre COBOL com CGI (Common Gateway Interface), oferecendo uma forma moderna de construir APIs REST e aplicações web mantendo a robustez do COBOL. Veja mais em docs/FRAMEWORK.md.

Controller Simples

app/Controllers/WelcomeController.cob Controller 
IDENTIFICATION DIVISION.
PROGRAM-ID. WelcomeController.

CLASS-ID. WelcomeController Extends BaseController.

* Responde a GET /welcome
METHOD-ID. index OVERRIDE.
    DISPLAY "Bem-vindo ao Laracol!"
    GOBACK.
END METHOD.

* Renderiza view de boas-vindas
METHOD-ID. show OVERRIDE.
    MOVE "welcome.index" TO WS-VIEW-NAME
    PERFORM RENDER-VIEW
    GOBACK.
END METHOD.

END CLASS WelcomeController.

Definição de Rotas REST

As rotas são definidas em COBOL e mapeiam URLs diretamente para métodos de Controllers.

routes/api.cbl Routing 
* Rotas REST mapeadas para Controllers
* Arquivo: routes/api.cbl

GET   /api/users           →  UserController::index
POST  /api/users           →  UserController::store
GET   /api/users/:id       →  UserController::show
PUT   /api/users/:id       →  UserController::update
DELETE /api/users/:id      →  UserController::destroy

Middleware de Autenticação

app/Middleware/AuthMiddleware.cob Middleware 
METHOD-ID. handle OVERRIDE.
    EVALUATE WS-AUTH-TOKEN
        WHEN SPACES
            MOVE 401 TO WS-HTTP-STATUS
            MOVE "Unauthorized" TO WS-RESPONSE-BODY
            PERFORM ABORT-REQUEST
        WHEN OTHER
            PERFORM VALIDATE-TOKEN
            PERFORM NEXT-MIDDLEWARE
    END-EVALUATE.
    GOBACK.
END METHOD.
