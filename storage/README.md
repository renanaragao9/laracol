# Storage | Armazenamento

📋 **Português BR:** Diretório para armazenar dados gerados pela aplicação como logs, uploads e cache.

📋 **English:** Directory to store data generated by the application such as logs, uploads, and cache.

---

## Português BR - Estrutura de Armazenamento

### O que vai em Storage?

Storage é usado para armazenar dados gerados em tempo de execução:

- **logs/**: Arquivos de log da aplicação
- **app/**: Arquivos enviados por usuários
- **cache/**: Arquivos de cache (opcional)
- **sessions/**: Dados de sessão (opcional)

### Estrutura do Diretório

```
storage/
├── app/         - Downloads e uploads
├── logs/        - Arquivos de log
├── cache/       - Cache de aplicação
└── README.md    - Este arquivo
```

### Exemplo de Logging em COBOL

```cobol
       IDENTIFICATION DIVISION.
       PROGRAM-ID. LOGSERVICE.

      *> Comentário: Serviço de logging
       DATA DIVISION.
       WORKING-STORAGE SECTION.

       01 LOG-FILE PIC X(100) VALUE
           "storage/logs/application.log".
       01 LOG-MESSAGE PIC X(500).
       01 LOG-LEVEL PIC X(20).
       01 TIMESTAMP PIC X(30).

       LINKAGE SECTION.
       01 MESSAGE PIC X(500).
       01 LEVEL PIC X(20).

       PROCEDURE DIVISION USING MESSAGE LEVEL.

      *> Comentário: Formata mensagem de log
           MOVE FUNCTION CURRENT-DATE TO TIMESTAMP
           MOVE MESSAGE TO LOG-MESSAGE
           MOVE LEVEL TO LOG-LEVEL

      *> Comentário: Escreve em arquivo de log
           DISPLAY "[" & FUNCTION TRIM(TIMESTAMP) & "] " &
               "[" & FUNCTION TRIM(LOG-LEVEL) & "] " &
               FUNCTION TRIM(LOG-MESSAGE)

           GOBACK.
```

### Níveis de Log

| Nível   | Descrição              | Uso                                    |
| ------- | ---------------------- | -------------------------------------- |
| DEBUG   | Informações detalhadas | Desenvolvimento                        |
| INFO    | Informações gerais     | Eventos importantes                    |
| WARNING | Avisos                 | Situações potencialmente problemáticas |
| ERROR   | Erros                  | Erros que não impedem execução         |
| FATAL   | Erros críticos         | Erros que impedem execução             |

### Exemplo de Log

Arquivo `storage/logs/application.log`:

```
[2024-03-13 10:30:45.123] [INFO] Aplicação iniciada
[2024-03-13 10:30:46.456] [DEBUG] Conectando ao banco de dados
[2024-03-13 10:30:47.789] [INFO] Banco de dados conectado
```

### Gerenciamento de Upload

```cobol
       IDENTIFICATION DIVISION.
       PROGRAM-ID. FILEUPLOADSERVICE.

      *> Comentário: Serviço para upload de arquivos
       DATA DIVISION.
       WORKING-STORAGE SECTION.

       01 UPLOAD-DIR PIC X(100) VALUE "storage/app/uploads".
       01 FILE-NAME PIC X(100).
       01 FILE-PATH PIC X(200).
       01 FILE-SIZE PIC 9(10).

       LINKAGE SECTION.
       01 UPLOADED-FILE PIC X(500).
       01 RESULT-PATH PIC X(200).

       PROCEDURE DIVISION USING UPLOADED-FILE RESULT-PATH.

      *> Comentário: Processa arquivo de upload
           MOVE UPLOADED-FILE TO FILE-NAME
           STRING UPLOAD-DIR DELIMITED BY SIZE
               "/" DELIMITED BY SIZE
               FILE-NAME DELIMITED BY SIZE
               INTO FILE-PATH
           END-STRING

           MOVE FILE-PATH TO RESULT-PATH

           GOBACK.
```

## English - Storage Structure

### What goes in Storage?

Storage is used for storing runtime-generated data:

- **logs/**: Application log files
- **app/**: Files uploaded by users
- **cache/**: Application cache (optional)
- **sessions/**: Session data (optional)

### Directory Structure

```
storage/
├── app/         - Downloads and uploads
├── logs/        - Log files
├── cache/       - Application cache
└── README.md    - This file
```

### Log Levels

| Level   | Description          | Usage                               |
| ------- | -------------------- | ----------------------------------- |
| DEBUG   | Detailed information | Development                         |

| WARNING | Warnings             | Potentially problematic situations  |
| ERROR   | Errors               | Errors that don't prevent execution |
| FATAL   | Critical errors      | Errors that prevent execution       |

## Permissões de Arquivo | File Permissions

Configure as permissões corretas:

Set the correct permissions:

```bash
# Português BR | Portuguese BR
chmod 755 storage/       # Diretório de storage
chmod 755 storage/logs/  # Diretório de logs
chmod 755 storage/app/   # Diretório de uploads

# English | American English
chmod 755 storage/       # Storage directory
chmod 755 storage/logs/  # Logs directory
chmod 755 storage/app/   # Uploads directory
```

## Rotação de Logs | Log Rotation

```cobol
       IDENTIFICATION DIVISION.
       PROGRAM-ID. LOGROTATIONSERVICE.

      *> Comment: Log rotation for old files
       DATA DIVISION.
       WORKING-STORAGE SECTION.

       01 LOG-DIRECTORY PIC X(100) VALUE "storage/logs".
       01 MAX-LOG-SIZE PIC 9(10) VALUE 10485760.
       01 ARCHIVE-DATE PIC X(20).

       PROCEDURE DIVISION.

      *> Comment: Check if logs need rotation
           MOVE FUNCTION CURRENT-DATE TO ARCHIVE-DATE

           DISPLAY "Checking logs for rotation..."
           DISPLAY "Archive date: " & ARCHIVE-DATE

           GOBACK.
```
